## Core Idea

`appendMeditateGitignore()` in `src/cli/commands/meditate.ts` writes `.mcp.ralph-*.json` to the project's `.gitignore`. The actual file generated by `agent.ts`'s `writeMcpConfig()` is named `.mcp-${this.config.name}-${Date.now()}.json` — for the meditate agent, `.mcp-meditate-1776052427716.json`. The gitignore pattern uses dots and a hardcoded "ralph" prefix. The real filename uses hyphens and the agent name. These never match. The file `.mcp-meditate-1776052427716.json` sitting untracked in the project root right now is direct evidence: it survived a session and shows up in `git status` as `??`.

## Why It Matters

The `finally` block in `agent.ts`'s `run()` calls `cleanupMcpConfig()` to delete the temp file. In normal operation this works. On abrupt kill (SIGKILL, OOM, power loss), `finally` does not run. The gitignore is the last-resort safety net for that case — but it's been wrong since it was written.

Every project that has ever run `ralph meditate` has `.mcp.ralph-*.json` in its `.gitignore`. None of those entries have ever matched a real file. The current ralph-cli `.gitignore` (line 8) carries this incorrect entry. Any project-level `.gitignore` written by `appendMeditateGitignore` also carries it.

The `appendMeditateGitignore` function hardcodes the pattern as a string literal. The agent naming convention is defined separately in `agent.ts` and has no connection to that literal. The two are already out of sync and will diverge again if an agent is renamed.

## Revised Implementation Steps

1. **Fix the gitignore pattern in `appendMeditateGitignore`.** Change `.mcp.ralph-*.json` to `.mcp-*-*.json`. This matches any file of the form `.mcp-{name}-{timestamp}.json` regardless of agent name. Update the entry in `src/cli/commands/meditate.ts`.

2. **Fix the ralph-cli root `.gitignore` directly.** Line 8 currently says `.mcp.ralph-*.json`. Change it to `.mcp-*-*.json`. Delete the orphaned `.mcp-meditate-1776052427716.json` file from the project root.

3. **Write one unit test in `src/cli/tests/meditate.test.ts` that calls `appendMeditateGitignore` on a temp dir and then asserts the resulting `.gitignore` contains `.mcp-*-*.json` (not the old pattern).** The existing tests for this function exist but did not catch the pattern mismatch because they likely only assert that the file contains some entries, not what those entries are.

4. **Derive the gitignore pattern from the same constant that drives `writeMcpConfig`.** In `agent.ts`, the filename is built from a template: `.mcp-${name}-${Date.now()}.json`. Export a helper `mcpConfigGlobPattern(): string` that returns `.mcp-*-*.json` (or parameterized by name if needed). Import it in `appendMeditateGitignore` so the two stay in sync. This is a one-line extraction but it ties the two sites together at the type level.
