## Findings

1. **What:** T0900's dispatched plan (`docs/superpowers/plans/2026-04-26-mark-plan-implemented-will-have-no-caller.md`) is underspecified. It adds `mark_plan_implemented` to memory-writer's MCP and rubric step 7, but 11 post-dispatch illuminations (T1000–T1900) evolved the diagnosis to a 3-change atomic fix the plan does not encode. Implementing the plan as written ships a partial fix while the live pipeline continues to skip both lifecycle closures on every run.

   **Evidence:**
   - T0900 plan step 3: "Add step 7 to the `memory_writer.md` procedure… 'Call `mark_plan_implemented`…'" — this assumes rubric step 7 fires.
   - `2026-04-26T1600-dual-procedure-problem-not-absent-rubric.md`: "The LLM now receives the 8-step rubric first, then the 6-step inline list from the node task after the separator… the later, shorter one… acts as the operative procedure." The plan does not delete the inline 6-step list that blocks step 7.
   - `2026-04-26T1800-plan-closes-illumination-stays-open.md`: "`mark_implemented` appears nowhere in `memory-writer.md`: not in the procedure, not in the tools whitelist." The plan does not add the illumination-side closer.

   **Why it matters:** A plan marked `implemented` after partial execution produces a false positive in the janitor reconciliation loop — T0900's illumination gets closed, confidence accumulates, but neither closure fires in the live pipeline. The T0100 dead-end state (`dispatched` forever) repeats silently under a different flag. Static grep (`grep mark_plan_implemented memory-writer.md`) returns a hit and looks correct; the failure is undetectable without running the assembled prompt.

   **Suggested action:** Expand the T0900 plan to add the three remaining changes described in `2026-04-26T1900-t0900-plan-closure-sits-in-confirmed-dead-zone.md`: (1) delete inline 6-step list from `memory_writer` node prompt in `pipelines/illumination-to-implementation.dot`, (2) add `mcp__illumination__mark_implemented` to `src/cli/agents/memory-writer.md` tools list, (3) add step 8 calling `mark_implemented` on `$illumination_path` with the same best-effort error policy as step 7. This collapses T1000–T1900 into a single commit and closes the open-close pair completely.
