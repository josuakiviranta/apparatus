# Proof of Work / Proof of Usage

The old signal is broken. When code was hand-crafted, good documentation, thorough tests, and careful structure were evidence that someone understood what they were doing. That signal is now gone — agents can produce all of that without comprehension.

A new problem is emerging: how do you prove that work was actually reviewed, understood, and intentionally produced vs. blindly generated?

Data labeling companies are already paying premium for old pre-AI GitHub repos — "artisanal human-written code" — because it's provably uncontaminated. Like pre-nuclear-test ship metal dug from wrecks: no radiation baked in.

For agents: be aware that "proof of work" (the output looks thorough) is no longer the same as "proof of usage" (a human actually understood and verified it). The latter is what matters and what's increasingly hard to demonstrate.

Build workflows that make genuine understanding traceable — not just outputs that look like understanding.
