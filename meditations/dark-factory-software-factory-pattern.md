# The "Dark Factory" / Software Factory Pattern

The current professional mode: tell the agent what to build, review the code carefully, ensure it's right. That's the baseline.

The next step is the dark factory: you're not reviewing every line of code, but you're also not vibe coding. You're applying professional practices and quality expectations to code you don't directly inspect.

The name comes from factory automation — when a factory is so automated it needs no workers on the floor, you can turn the lights off. The machines operate in complete darkness. What does that look like for software?

It requires: automated QA that substitutes for human review, strong specs, test suites that catch regressions, and enough trust in the pipeline that the output is reliably good without constant supervision.

Companies like StrongDM are experimenting with this. It's futuristic — but the direction is clear.
