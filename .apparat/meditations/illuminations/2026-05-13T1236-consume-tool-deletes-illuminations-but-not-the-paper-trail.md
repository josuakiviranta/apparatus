## Core Idea

ADR-0002 (`docs/adr/0002-consume-only-illumination-lifecycle.md`) collapsed the illumination lifecycle to two states — alive in `.apparat/meditations/illuminations/` or `git rm`'d via `consume(filename, reason)`. The same ADR went further: it deliberately severed the slug-link between illumination and downstream artefacts ("Plan files no longer carry `illumination_source` frontmatter; filename slug-mirroring between illumination and plan is incidental, not relied upon"). One cycle of `illumination-to-implementation` lays down five artefacts — illumination + spec + plan + review + session — but the lifecycle covers only the first. The other four pile up forever. Today's working tree: 14 alive illuminations, **21 plans + 38 specs + 6 reviews + 52 sessions = 117 design-trail files** with no parent illumination still present for most of them.

The `consume` MCP tool is a shallow module: tiny interface, tiny implementation. Its parent ADR is exactly the wrapper-that-hides-nothing anti-pattern called out in `.apparat/meditations/stimuli/deep-modules-hide-complexity.md` — the lifecycle is implemented for one artefact type with no seam forcing the other four to agree.

## Why It Matters

Three concrete symptoms verify this is not theoretical:

1. **Orphans by example.** `docs/superpowers/plans/2026-04-30-consume-only-illumination-lifecycle.md` is the plan for ADR-0002 itself. Its parent illumination was consumed (no file matching that slug in `.apparat/meditations/illuminations/`). The plan that built the lifecycle outlives the lifecycle it was meant to ride. Same pattern repeats for ~21 plan files older than the current illumination cohort.

2. **Three existing illuminations attack symptoms of this single missing seam.** `2026-05-12T1028-collapse-memory-tail-and-tier-pipeline-models.md` says "sessions/ has 48 entries and zero readers — collapse the tail." `2026-05-12T2255-doc-drift-tail-for-parallel-implementation.md` says "kill the dead `.apparat/sessions/` write entirely." `2026-05-13T1106-adr-0015-failure-half-has-no-reaper-and-sessions-misclassified.md` says "the asymmetric-GC ADR needs a second pass; sessions are not institutional memory." Each treats a different downstream type as a special-case write-only path. The pattern under all three: **ADR-0002 stopped at one artefact type, and every subsequent pipeline that produces a design-trail file inherits the no-lifecycle gap**.

3. **The severance was deliberate, costed at N=1.** ADR-0002 at the time had one downstream artefact type to consider (plans). Today's `illumination-to-implementation/pipeline.dot` has five (`design-writer.md` → spec, `plan-writer.md` → plan, `review_gate.md` → review, `memory-writer.md` → session, `memory-reflector.md` → reflection-summary). The "incidental, not relied upon" language costed slug-mirroring at one fork; today's N=5 reverses the cost calculation — the slug is the only seam available without re-introducing frontmatter coupling.

Per `VISION.md`, this is the single-developer harness for managing many projects with many agents; the `.apparat/` partition principle in ADR-0007/0008 said apparat-owned scratch lives under `.apparat/`. Sessions are apparat-owned scratch by that rule and should never have been treated as "institutional memory" (ADR-0015's own claim was contradicted within 24 hours per `2026-05-13T1106`). Specs/plans/reviews live under `docs/superpowers/` because they were once thought to be the third-party doc convention — but no third party reads them; only the in-flight cycle does. By the same partition logic they are cycle-scratch.

The deepening move: one `consume(illumination_slug, reason)` invocation walks all five locations (`.apparat/meditations/illuminations/<slug>.md`, `docs/superpowers/specs/*-<slug>-design.md`, `docs/superpowers/plans/*-<slug>.md`, `docs/superpowers/reviews/*-<slug>-review.md`, `.apparat/sessions/*-<slug>.md`) and `git rm`'s each match in a single commit. The consume interface stays narrow; everything else moves behind it. Locality: lifecycle change concentrates in `consume.mjs`. Leverage: deleting one illumination cleans its entire paper trail. Seam: filename-slug is restored as the explicit cross-artefact agreement, with a single test asserting that every `git rm` consume command resolves to N≥1 matches.

This subsumes — does not duplicate — the three prior illuminations: rather than killing the memory-writer + memory-reflector nodes (2026-05-12T1028 / T2255), it makes their output participate in the same disposable lifecycle illuminations already have. Rather than building a separate `apparat gc` sweep command (2026-05-13T0805), it extends an existing primitive that already runs on every successful implement. Notes are all marked `[x]` so the steer "focus on notes.md if not already marked" is null this round; this illumination is anchored on tree evidence + ADR-0002 + the cited prior illuminations, not on any open note.

## Revised Implementation Steps

1. **Audit slug-collision risk before changing anything.** Write a one-off script (or run from `consume.mjs`) that scans `.apparat/meditations/illuminations/` and reports, for each alive illumination's slug, how many matches exist in `docs/superpowers/{specs,plans,reviews}/` and `.apparat/sessions/`. Confirm no slug collides across illuminations (current 14 are all `YYYY-MM-DDThhmm-<slug>` so the slug portion alone should be unique; verify).

2. **Extend `consume.mjs` in `src/cli/pipelines/illumination-to-implementation/` (and the parallel sibling).** Today `consume(filename, reason)` runs `git rm <illumination_path>`. Add four glob deletions in the same commit: `docs/superpowers/specs/*-<slug>-design.md`, `docs/superpowers/plans/*-<slug>.md`, `docs/superpowers/reviews/*-<slug>-review.md`, `.apparat/sessions/*-<slug>.md`. Use the slug portion (everything after `YYYY-MM-DDThhmm-` and before `.md`) — exact regex documented inline. Each glob may match 0 or 1; matching 0 is fine (cycle never produced that artefact). The single commit message becomes `meditate: consume <filename> (<reason>) [+N trail files]`.

3. **Update ADR-0002 with an addendum.** Add a `## 2026-05-13 addendum` section reversing the "filename slug-mirroring is incidental" line: slug is now the cross-artefact seam, enforced by `consume.mjs` and by a new `consume-deletes-paper-trail` smoke test. Cite the N=5 today vs N=1 cost calculation. Keep the historical text intact above the addendum.

4. **Add a smoke scenario `.apparat/scenarios/consume-deletes-paper-trail/`** that seeds a fake illumination + a matching `-design.md` + matching plan + matching session, invokes `consume`, and asserts all four files are gone in the resulting commit. Keep it parameterised so the next artefact type the pipeline adds drops in one fixture line.

5. **Backfill cleanup — one-shot, separately committed.** Write a `scripts/backfill-consume-orphans.mjs` that lists every file in `docs/superpowers/{plans,specs,reviews}/` and `.apparat/sessions/` whose slug has no matching alive illumination, prints the proposed delete list for human review, then on `--apply` git-rm's the lot in one commit. This is the equivalent of ADR-0015's retroactive-cleanup that was explicitly deferred — extending the same idea to the upstream design trail. Run it once to clear the existing 117 files (after sanity-reviewing the printed list).

6. **Remove the deferred-collapse work from the queue.** Once steps 2–5 ship, the three prior illuminations (`2026-05-12T1028`, `2026-05-12T2255`, `2026-05-13T1106`) downgrade from "kill the memory tail" to "memory tail is fine, its output rides the lifecycle now." Consume them with reason `superseded-by-consume-lifecycle-extension`. This is the cheapest possible outcome — instead of editing two `pipeline.dot` files and rewriting agent rubrics, one MCP tool change subsumes all three.
