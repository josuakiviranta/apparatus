# Start Every Project with a Thin Boilerplate Template

Coding agents are phenomenally good at matching existing style when there's something to match. Give them a starting point.

Don't write paragraphs in CLAUDE.md describing how you like to work. Instead, start with a very thin skeleton — just enough hints on structure, naming, and patterns — and the agent will pick it up and roll with it.

The template doesn't need to be big. It's a minimal empty structure that encodes your preferences implicitly through the code itself rather than through instructions. The agent reads the code, not just the docs.

This is how you get consistent style across an entire project without constant correction.
