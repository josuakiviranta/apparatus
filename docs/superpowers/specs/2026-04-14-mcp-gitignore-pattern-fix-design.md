# MCP Config Gitignore Pattern Fix Design

**Date:** 2026-04-14
**Status:** Approved
**Feature:** Fix mismatched gitignore pattern for MCP config files

## Problem

`appendMeditateGitignore()` in `src/cli/commands/meditate.ts` writes `.mcp.ralph-*.json` to the project's `.gitignore`. The actual MCP config files generated by `agent.ts`'s `writeMcpConfig()` use the template `.mcp-${this.config.name}-${Date.now()}.json`, producing filenames like `.mcp-meditate-1776156013597.json`.

The gitignore pattern uses dots and a hardcoded "ralph" prefix. The real filenames use hyphens and the agent name as prefix. These never match. On abrupt kill (SIGKILL, OOM), the `finally` cleanup doesn't run, and the gitignore safety net fails — orphaned MCP config files show up as untracked in `git status`.

The root `.gitignore` of ralph-cli itself (line 7) carries the same wrong pattern.

## Solution

1. Fix the gitignore pattern to `.mcp-*-*.json`
2. Extract a shared constant so the pattern and the filename template stay in sync
3. Fix the root `.gitignore`

## Architecture

### Shared constant: `src/attractor/core/agent.ts`

Export a constant from the file that already owns the filename template:

```typescript
/** Glob pattern matching MCP config files generated by writeMcpConfig() */
export const MCP_CONFIG_GLOB = ".mcp-*-*.json";
```

The existing filename construction `.mcp-${this.config.name}-${Date.now()}.json` already lives here. Colocating the glob pattern ensures any future rename updates both.

### Consumer: `src/cli/commands/meditate.ts`

Import `MCP_CONFIG_GLOB` from `agent.ts`. Replace the hardcoded `.mcp.ralph-*.json` string literal in `appendMeditateGitignore()` with the imported constant.

### Root `.gitignore`

Change line 7 from `.mcp.ralph-*.json` to `.mcp-*-*.json`. Delete the orphaned `.mcp-meditate-1776156013597.json` from the project root.

## Data Flow

```
agent.ts writeMcpConfig()
  filename: `.mcp-${name}-${timestamp}.json`
                    |
                    v
agent.ts MCP_CONFIG_GLOB = ".mcp-*-*.json"  <-- single source of truth
                    |
                    v
meditate.ts appendMeditateGitignore()
  writes MCP_CONFIG_GLOB to project .gitignore
                    |
                    v
git status: orphaned files matched, not shown as untracked
```

## Constraints

- The glob `.mcp-*-*.json` is intentionally broad — matches any agent name. This is correct: the gitignore should catch orphans from any agent, not just "meditate".
- No new files created. The constant lives in `agent.ts` where the filename template already exists.
- Import path from `meditate.ts` to `agent.ts`: use the existing relative path convention already used in the codebase.

## Out of Scope

- Migrating existing project `.gitignore` files that already have the wrong pattern — users must fix those manually or re-run `ralph meditate` (which will append the correct pattern)
- A CLI migration command to fix old patterns — YAGNI
- Parameterizing the glob by agent name — YAGNI, broad match is safer
