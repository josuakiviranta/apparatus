# MCP Config Gitignore Pattern Fix Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the mismatched gitignore pattern so orphaned MCP config files are correctly ignored by git.

**Architecture:** Export a `MCP_CONFIG_GLOB` constant from `agent.ts` (where the filename template lives), import it in `meditate.ts` to replace the hardcoded wrong pattern, fix the root `.gitignore`, and delete the orphaned file.

**Tech Stack:** TypeScript, vitest

### Task 2: Update `appendMeditateGitignore` to use the constant

**Files:**
- Modify: `src/cli/commands/meditate.ts:46`
- Modify: `src/cli/tests/meditate.test.ts:128`

- [ ] **Step 1: Update the failing test assertion**

In `src/cli/tests/meditate.test.ts`, find the test `"adds .mcp.ralph-*.json to .gitignore"` (around line 128). Change:

```typescript
// Before:
it("adds .mcp.ralph-*.json to .gitignore", () => {
  appendMeditateGitignore(tmpDir);
  const content = readFileSync(join(tmpDir, ".gitignore"), "utf8");
  expect(content).toContain(".mcp.ralph-*.json");
});

// After:
it("adds .mcp-*-*.json to .gitignore", () => {
  appendMeditateGitignore(tmpDir);
  const content = readFileSync(join(tmpDir, ".gitignore"), "utf8");
  expect(content).toContain(".mcp-*-*.json");
});
```

- [ ] **Step 2: Run test to verify it fails**

Run: `npx vitest run src/cli/tests/meditate.test.ts -t "adds .mcp"`
Expected: FAIL — `.mcp-*-*.json` not found in gitignore content (old pattern `.mcp.ralph-*.json` is written instead)

- [ ] **Step 3: Update `meditate.ts` to import and use the constant**
